# Skill: telegram-bot-development

Разработка и создание Telegram-ботов: от простого бота до ботов с меню, командами, inline-кнопками и интеграциями.

## Когда использовать

- Пользователь просит создать Telegram-бота, доработать бота, добавить команды или сценарии.
- Нужна архитектура бота, выбор стека (Node.js / Python), webhook vs long polling.
- Работа с Telegram Bot API: сообщения, клавиатуры, callback_query, медиа, документы.

## Задача агента

- Выбирать стек: **Node.js** (grammY, node-telegram-bot-api, telegraf) или **Python** (python-telegram-bot, aiogram).
- Проектировать структуру: команды, обработчики, контекст/состояние (session, FSM).
- Реализовывать: /start, меню, inline-кнопки, приём текста/фото/документов, сохранение данных (БД или Google Sheets — см. skill **google-sheets**).
- Учитывать лимиты API (длина сообщения 4096, rate limits), безопасность (не логировать токен, использовать env).

## Базовый стек

- **Node.js**: grammY (рекомендуется), telegraf, node-telegram-bot-api.
- **Python**: python-telegram-bot (v20+), aiogram 3.x.
- **Документация**: Telegram Bot API — получать через Context7 MCP при необходимости.

## Типовые сценарии

1. **Простой бот с командами** — /start, /help, ответ на текст; без БД.
2. **Бот с меню и кнопками** — ReplyKeyboardMarkup, InlineKeyboardMarkup, callback_query.
3. **Бот с состояниями (FSM)** — многошаговые формы (опрос, заявка), хранение состояния по user/chat.
4. **Бот с данными** — запись заявок/ответов в Google Таблицы, БД или API (интеграция с **google-sheets**).
5. **Уведомления и рассылки** — отправка по расписанию или по событию; учёт rate limit.

## Рекомендации

- Токен и секреты — только в переменных окружения (.env), не в коде.
- Webhook для продакшена (HTTPS обязателен); long polling — для локальной отладки.
- Обрабатывать ошибки и повторные запросы (retry); логировать ошибки API.
- Для сложных диалогов — явная машина состояний (FSM) или сцены (grammY sessions, aiogram FSM).

## Результат

- Рабочий бот с понятной структурой, готовый к запуску и при необходимости к интеграции с таблицами/API.

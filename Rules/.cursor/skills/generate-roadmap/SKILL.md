# Skill: generate-roadmap

Создание пошагового плана разработки в файле `roadmap.md`.

## Когда использовать

- Документ **project-spec.md** создан (skill **generate-project-spec** выполнен).
- Нужно разбить разработку на этапы перед началом кода.

## Задача агента

Создать в корне проекта файл **roadmap.md**. План должен разбивать разработку на этапы и конкретные шаги (чек-листы), опираясь на project-spec.

## Обязательные этапы roadmap

1. **Инициализация проекта**
   - Репозиторий, окружение, зависимости, конфигурация.

2. **Создание структуры файлов**
   - Папки, базовые файлы (HTML/CSS/JS или по стеку из spec), точки входа.

3. **Верстка базовых компонентов**
   - Макет главной/ключевых страниц, типографика, сетка, компоненты UI.

4. **Реализация функционала**
   - Формы, навигация, интерактив, логика по spec.

5. **Интеграции**
   - API, аналитика, CMS, внешние сервисы из project-spec.

6. **Оптимизация**
   - Производительность, доступность, SEO (если применимо).

7. **Тестирование**
   - Проверка сценариев, устройств, кроссбраузерность.

## Формат roadmap.md

- Заголовки по этапам (как выше).
- Под каждым этапом — список задач в виде чекбоксов `- [ ]`.
- Задачи формулировать конкретно (например: «Создать компонент шапки с меню», «Подключить форму к CRM»).
- Учитывать специфику проекта из project-spec (лендинг / многостраничник / сервис).

## Результат

- Файл `roadmap.md` в корне проекта.
- После создания roadmap агент может приступать к реализации по этому плану (создание структуры → верстка → функционал и т.д.).

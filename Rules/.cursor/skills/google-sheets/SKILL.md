# Skill: google-sheets

Работа с Google Таблицами: чтение и запись данных, формулы, форматирование, интеграция с ботом или приложением.

## Когда использовать

- Пользователь просит работать с Google Таблицами: выгрузить данные, записать заявки, построить отчёт.
- Нужна интеграция бота (Telegram и др.) с таблицей — запись лидов, ответов, логов.
- Нужны скрипты в Google Apps Script (макросы, триггеры, кастомные функции в ячейках).

## Задача агента

- **Через API (внешнее приложение)**:
  - Google Sheets API v4: аутентификация (OAuth2 или сервисный аккаунт), чтение диапазонов (A1-нотация), запись, добавление строк.
  - Библиотеки: Node.js — `googleapis`; Python — `gspread`, `google-auth` + Sheets API.
- **Внутри Google (Apps Script)**:
  - Скрипты в таблице: `SpreadsheetApp`, чтение/запись ячеек и диапазонов, создание листов, форматирование.
  - Триггеры: по времени, при редактировании; кастомные функции в ячейках.
- Учитывать квоты (лимиты запросов), не делать избыточных вызовов при больших объёмах.

## Базовый стек

- **Node.js**: пакет `googleapis` (или обёртки типа `google-spreadsheet`).
- **Python**: `gspread` + учётные данные (сервисный аккаунт JSON или OAuth).
- **Google Apps Script**: встроенная среда в таблице (JavaScript), деплой как веб-приложение или триггеры.

## Типовые сценарии

1. **Чтение данных из таблицы** — получить диапазон (например, список контактов, справочников), вернуть в коде или боту.
2. **Запись заявок из бота** — Telegram-бот (skill **telegram-bot-development**) получает данные пользователя → добавление строки в таблицу (имя, контакт, дата).
3. **Отчёты и сводки** — агрегация данных по листам/диапазонам, формирование итоговой таблицы или выгрузка.
4. **Автоматизация в Apps Script** — триггер «раз в день» обновляет лист; кнопка в таблице запускает скрипт; формула в ячейке вызывает `=MY_FUNCTION()`.
5. **Форматирование** — заголовки, ширина столбцов, условное форматирование через API или Apps Script.

## Учётные данные и доступ

- **Сервисный аккаунт**: JSON-ключ, шаринг таблицы на email сервисного аккаунта («редактор» или «чтение»).
- **OAuth2**: для доступа от имени пользователя (если нужны «его» таблицы).
- Токены и ключи — в переменных окружения, не в репозитории.

## Результат

- Корректное чтение/запись в Google Таблицы из кода или через Apps Script; при интеграции с ботом — согласованная структура таблицы (заголовки, типы данных) и обработка ошибок доступа/квот.

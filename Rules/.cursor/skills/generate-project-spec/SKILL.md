# Skill: generate-project-spec

Преобразование ответов интервью в структурированный документ технического описания проекта `project-spec.md`.

## Когда использовать

- Интервью проведено (skill **project-interview** выполнен) и ответы получены.
- Нужно зафиксировать спецификацию перед созданием roadmap и кодом.

## Задача агента

Создать в корне проекта файл **project-spec.md** со следующей структурой. Заполнить разделы на основе ответов пользователя из интервью.

## Структура project-spec.md

```markdown
# Техническое описание проекта

## 1. Описание проекта
Краткое описание: что за проект, для кого, зачем.

## 2. Цели проекта
- Цель 1
- Цель 2
- …

## 3. Целевая аудитория
- Сегменты, демография, ключевые потребности и боли.

## 4. Основные функции
- Список страниц, блоков, форм, интеграций.
- Приоритеты (must have / nice to have).

## 5. Архитектура сайта
- Структура страниц (sitemap).
- Навигация, ключевые сценарии пользователя.

## 6. Технологический стек
- Frontend, backend (если есть), CMS, хостинг, инструменты.
- Ограничения и предпочтения по стеку.
```

## Правила

- Текст должен быть конкретным: без «например» и размытых формулировок, где можно взять данные из интервью.
- Если по какому-то пункту в интервью не было ответа — пометить в spec как «Уточнить у заказчика» или оставить подзаголовок с пустым списком.
- После создания project-spec агент должен перейти к skill **generate-roadmap** и создать `roadmap.md`.

## Результат

- Файл `project-spec.md` в корне проекта.
- Основа для формирования roadmap и дальнейшей разработки.

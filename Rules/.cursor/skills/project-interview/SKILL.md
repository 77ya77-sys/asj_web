# Skill: project-interview

Проведение интервью с пользователем перед началом любого нового проекта.

## Когда использовать

- Пользователь говорит о новом проекте, лендинге, сайте или сервисе.
- Нет документа `project-spec.md` в корне проекта.
- Нужно уточнить цели, аудиторию и требования до написания кода.

## Задача агента

Провести структурированное интервью, задав вопросы по следующим блокам. Не писать код до получения ответов и формирования project-spec.

### Блоки вопросов

1. **Тип проекта**
   - Лендинг, многостраничный сайт, веб-сервис, приложение?

2. **Цель проекта**
   - Что должен достичь проект? (продажи, лиды, информирование, сервис)

3. **Целевая аудитория**
   - Кто основные пользователи? Возраст, интересы, боли.

4. **Регион работы**
   - География (РФ, СНГ, мир), язык интерфейса, локаль.

5. **Основные функции сайта**
   - Какие страницы, блоки, формы, интеграции нужны?

6. **Способ монетизации**
   - Реклама, подписка, продажа товаров/услуг, лиды, другое.

7. **Технические предпочтения (стек)**
   - HTML/CSS/JS, React, Next.js, CMS, хостинг, особые требования.

## Результат

- Собранные ответы пользователя в структурированном виде.
- Передача контекста skill'у **generate-project-spec** для формирования документа `project-spec.md`.

## Поведение

- Задавать вопросы по одному блоку или группой, не перегружая одним сообщением.
- Уточнять неясные ответы.
- После сбора ответов — вызвать/применить skill **generate-project-spec** и создать `project-spec.md`.

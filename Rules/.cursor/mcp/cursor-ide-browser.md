# Cursor IDE Browser MCP

**Идентификатор:** `cursor-ide-browser`  
**Статус:** подключён в проекте (доступен агенту)

## Назначение

Браузер внутри IDE: снимки DOM, взаимодействие с элементами, формы, скролл, профилирование производительности.

## Основные возможности

- **browser_navigate** — открытие URL.
- **browser_snapshot** — снимок структуры страницы и refs для кликов/ввода.
- **browser_click**, **browser_type**, **browser_fill** — взаимодействие с элементами.
- **browser_scroll**, **browser_tabs** — скролл и вкладки.
- **browser_profile_start/stop** — CPU-профилирование (поиск узких мест).
- **browser_handle_dialog** — обработка alert/confirm/prompt.

## Важно

- Сначала **browser_navigate**, затем **browser_lock** перед действиями, в конце **browser_unlock**.
- Перед кликом/вводом делать **browser_snapshot** и использовать ref из снимка.
- Для тяжёлых страниц — короткие ожидания (1–3 с) и повторные snapshot.

## Когда применять

- Тестирование и отладка фронтенда и лендингов.
- Проверка вёрстки и интерактивности.
- Поиск причин тормозов (профилирование).
